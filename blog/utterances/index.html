¿Cómo configuro Utterances?

Para configurar Utterances, un sistema de comentarios que utiliza problemas de GitHub para comentarios, para su sitio web o aplicación web, debe seguir estos pasos:

  1. Elija el repositorio: Utterances requiere un repositorio de GitHub para almacenar los comentarios. Puede usar un repositorio existente o crear uno nuevo. El repositorio debe ser público y debe tener acceso de escritura.
  2. Genere un token de acceso: para permitir que Utterances acceda al repositorio, debe generar un token de acceso personal de GitHub. Para generar el token, vaya a la configuración de su cuenta de GitHub, haga clic en "Configuración de desarrollador" > "Tokens de acceso personal" y luego haga clic en "Generar nuevo token". Asigne un nombre al token, seleccione el alcance "repo" y luego haga clic en "Generar token". Copie el token en un lugar seguro, ya que no podrá volver a verlo.
  3. Instale Utterances: puede instalar Utterances en su aplicación Nuxt.js 3 agregando el script de Utterances a su archivo nuxt.config.js. Puede hacerlo agregando el siguiente código a su archivo nuxt.config.js:
export default {
  head: {
    script: [
      {
        src: "https://utteranc.es/client.js",
        async: true,
        "data-utterances-repo": "<USERNAME>/<REPOSITORY>",
        "data-utterances-theme": "<THEME>",
        "data-utterances-label": "<LABEL>",
        "data-utterances-issue-term": "<ISSUE-TERM>",
        "data-utterances-cleanup": "<CLEANUP>",
      },
    ],
  },
};

Reemplace los siguientes marcadores de posición:

  • <NOMBRE DE USUARIO>/<REPOSITORIO>: el nombre de usuario y el nombre del repositorio de GitHub que creó en el paso 1.
  • <THEME>: El nombre del tema de Utterances que desea utilizar. Puede usar el tema predeterminado configurando esto en github-light o github-dark, o puede crear su propio tema siguiendo las instrucciones en la documentación de Utterances.
  • <ETIQUETA>: (Opcional) Una etiqueta para agregar a cada problema creado por Utterances. Esto se puede usar para diferenciar entre problemas creados por Expresiones y otros problemas en el repositorio.
  • <ISSUE-TERM>: (Opcional) El término a utilizar al crear nuevos problemas. Puede usar el valor predeterminado de ruta de acceso para crear un hilo de problema separado para cada página de su sitio web.
  • <CLEANUP>: (Opcional) Si cerrar automáticamente los problemas que no han tenido nuevos comentarios en un cierto período de tiempo. Puede establecer esto en verdadero o falso. De forma predeterminada, se establece en falso.
  1. Habilite la aplicación Utterances: para permitir que Utterances publique comentarios en el repositorio, debe agregar la aplicación Utterances al repositorio. Para hacer esto, vaya a la configuración del repositorio, haga clic en "Aplicaciones GitHub instaladas" y luego haga clic en "Configurar" junto a la aplicación Utterances. Otorgue a la aplicación acceso al repositorio.

Después de completar estos pasos, debería poder ver el cuadro de comentarios Expresiones en su sitio web o aplicación web. Los visitantes pueden publicar comentarios usando su cuenta de GitHub y los comentarios se almacenarán como problemas en el repositorio configurado.
